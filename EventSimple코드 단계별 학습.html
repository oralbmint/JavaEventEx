📚 EventSimple 코드 단계별 학습

💻 소스 코드

public class EventSimple extends JFrame {
    public EventSimple() throws HeadlessException {
        super();
        this.setTitle("event simple");
        this.setSize(300, 100);
        this.setDefaultCloseOperation(EXIT_ON_CLOSE);
        
        Container contentPane = this.getContentPane();
        contentPane.setLayout(new FlowLayout());
        JButton button1 = new JButton("Click!");
        contentPane.add(button1);
        
        MyActionListener listener = new MyActionListener();
        button1.addActionListener(listener);
        
        this.setVisible(true);
    }
    
    class MyActionListener implements ActionListener {
        public void actionPerformed(ActionEvent e) {
            JButton btn = (JButton) e.getSource();
            if (btn.getText().equals("Click!")) {
                btn.setText("Clicked!");
            } else {
                btn.setText("Click!");
            }
        }
    }
    
    public static void main(String[] args) {
        new EventSimple();
    }
}

📖 단계별 설명

🏗️ 1단계: 클래스 선언

상속 관계: EventSimple이 JFrame을 상속받아 윈도우 기능을 가집니다.

• JFrame: 스윙의 기본 윈도우 클래스

• 상속을 통해 윈도우의 모든 기능을 자동으로 획득

• 창 크기, 제목, 닫기 버튼 등의 기본 기능 포함

🔧 2단계: 생성자 - 기본 설정

윈도우 초기화: 프레임의 기본 속성들을 설정합니다.

• setTitle(): 윈도우 제목 설정

• setSize(): 윈도우 크기 설정 (가로 300px, 세로 100px)

• setDefaultCloseOperation(): X 버튼 클릭 시 프로그램 종료

🎨 3단계: GUI 컴포넌트 생성

컴포넌트 배치: 버튼을 생성하고 윈도우에 추가합니다.

• getContentPane(): 컴포넌트를 추가할 컨테이너 획득

• FlowLayout: 컴포넌트를 순서대로 배치하는 레이아웃

• JButton: 클릭 가능한 버튼 컴포넌트 생성

• add(): 버튼을 컨테이너에 추가

🔗 4단계: 이벤트 리스너 등록

Observer 패턴: 버튼에 리스너를 등록하여 이벤트를 감지합니다.

• MyActionListener: 내부 클래스로 정의된 리스너

• addActionListener(): 버튼에 리스너 등록

• 버튼 클릭 시 리스너의 actionPerformed() 메소드가 자동 호출

⚡ 5단계: 이벤트 처리 로직

이벤트 핸들링: 버튼 클릭 시 실행될 코드를 구현합니다.

• ActionListener 인터페이스 구현

• e.getSource(): 이벤트를 발생시킨 컴포넌트 참조

• getText(): 버튼의 현재 텍스트 확인

• setText(): 조건에 따라 버튼 텍스트 변경

• 토글 기능: "Click!" ↔ "Clicked!" 반복

🚀 6단계: 프로그램 시작점

애플리케이션 실행: JVM이 가장 먼저 호출하는 메소드입니다.

• main(): 자바 프로그램의 진입점

• new EventSimple(): 객체 생성과 동시에 생성자 실행

• 생성자에서 GUI 초기화 및 이벤트 리스너 등록 완료

🎯 학습 목표

이 예제를 통해 다음을 이해할 수 있습니다:

  • 이벤트 드리븐 프로그래밍: 사용자 액션에 반응하는 프로그램
  • Observer 패턴: 리스너를 통한 이벤트 감지
  • 내부 클래스: 외부 클래스의 멤버에 접근 가능
  • 인터페이스 구현: ActionListener 인터페이스
